# 版本更新记录

本项目采用语义化版本号：`主版本.次版本.修订版本`。

- 主版本：产品定位、数据模型或使用方式发生重大变化；
- 次版本：新增兼容能力或完整业务模块；
- 修订版本：缺陷修复、规则调整和文档优化。

## v1.2.0 — 2026-07-23

### 版本主题

新增 **线上运行可观测性与故障排查体系**，让每日任务不仅生成日报，还能够回答是否运行、哪个步骤失败、数据量是否异常、Notion 是否成功以及应该如何排查。

### 新增

- 新增 `scripts/run_daily_ops.py` 运维执行器；
- 将单元测试、通用情报 Pipeline 和竞品采购雷达统一纳入一次运行；
- 每个步骤记录：
  - 开始时间和结束时间；
  - 执行命令；
  - 退出码；
  - 执行耗时；
  - stdout 和 stderr；
  - 失败错误摘要；
- 新增结构化运行摘要：
  - `data/run_logs/latest_run.json`；
  - `data/run_logs/latest_run.md`；
  - `data/run_logs/YYYY-MM-DD/<run_id>/run_summary.json`；
  - `data/run_logs/YYYY-MM-DD/<run_id>/run_summary.md`；
- 新增三类逐步骤日志：
  - `01_unit-tests.log`；
  - `02_general-pipeline.log`；
  - `03_competitive-radar.log`；
- 新增自动诊断规则：
  - 步骤执行失败；
  - 业务 Summary 缺失；
  - 通用采集为 0；
  - 竞品信号为 0；
  - 重点竞品覆盖率低于 30%；
  - 通用或竞品 Notion 同步失败；
- 新增 `success`、`warning`、`failed` 三级运行状态；
- 新增 GitHub Actions Job Summary，线上直接查看步骤状态和核心指标；
- GitHub Actions Artifact 增加 `data/run_logs/`，失败时仍通过 `if: always()` 上传证据；
- 新增日志密钥脱敏，覆盖 `NOTION_TOKEN`、`OPENAI_API_KEY` 和 `LLM_API_KEY`；
- 新增 5 项运维专项测试；
- 新增《运行与故障排查手册》；
- 新增《故障排查记录》及标准事件模板；
- 新增 `data/run_logs/README.md` 目录说明。

### 调整

- GitHub Actions 从分别执行三个步骤，调整为统一调用 `scripts/run_daily_ops.py`；
- macOS `scripts/run_daily.sh` 同样通过运维执行器运行；
- 工作流超时时间由 40 分钟调整为 45 分钟；
- 运行日志不写入 Git 历史，通过 Artifact 保留 30 天；
- 长期故障结论写入 `docs/故障排查记录.md`，避免仓库保存大量原始日志。

### 输出目录

```text
data/run_logs/
  latest_run.json
  latest_run.md
  YYYY-MM-DD/
    <run_id>/
      01_unit-tests.log
      02_general-pipeline.log
      03_competitive-radar.log
      run_summary.json
      run_summary.md
```

### 验证

- 运维执行器 Python 语法检查通过；
- 5 项运维专项测试通过；
- 原有 9 项竞品采购测试继续纳入每日测试发现；
- 工作流在任务失败时仍执行 Artifact 上传；
- 日志中的已配置密钥值会替换为 `***REDACTED***`。

## v1.1.0 — 2026-07-23

### 版本主题

新增 **AI 竞品与采购情报雷达**，将原有通用 AI/Agent 资讯收集能力扩展为可服务于企业 AI 产品、商业分析、采购机会和职业研究的结构化情报模块。

### 新增

- 新增独立模块 `src/competitive_procurement.py`；
- 新增十类情报数据源框架：
  - 政策和监管变化；
  - 行业研究报告；
  - 竞品官网和报价；
  - 招聘信息；
  - 招标采购平台；
  - 行业会议和展会；
  - 客户公开年报；
  - 搜索趋势和社交内容；
  - 渠道和合作伙伴；
  - 技术发展；
- 新增 `config/competitive_procurement.json`，配置类别、关键词、证据权重、采购阶段和数据源；
- 新增 `config/competitor_watchlist.json`，固定追踪 10 个企业 AI 竞品：
  1. Microsoft Copilot Studio；
  2. Salesforce Agentforce；
  3. ServiceNow AI Agents；
  4. SAP Joule Studio；
  5. UiPath Agentic Automation；
  6. Palantir AIP；
  7. Dify Enterprise；
  8. 扣子企业版；
  9. 阿里云百炼 / Model Studio；
  10. 腾讯云智能体开发平台 ADP；
- 每个竞品新增定位、个人相关性、官网、价格页、别名、关键词、独立动态源和观察维度；
- 新增竞品别名匹配和信号归属；
- 新增采购阶段识别：`intent`、`tender`、`award`、`partnership`；
- 新增人民币和美元预算提取，并统一输出 `budget_cny`；
- 新增情报评分、可信度、推荐动作和去重；
- 新增竞品覆盖率，日报固定展示 10 个竞品，未命中时显示 0 条；
- 新增 JSONL、Markdown 日报和 Summary 输出；
- 新增竞品日报 Notion 子页面同步；
- 新增 GitHub Actions 云端每日任务，北京时间 08:30 自动执行；
- 本地 `run_daily.sh` 增加竞品采购雷达调用；
- 新增 GitHub Artifact，运行结果保留 30 天；
- 新增 9 项核心单元测试；
- 新增《AI 竞品与采购情报雷达 PRD》和完整使用说明。

### 调整

- 明确竞品选择从“模型能力榜单”调整为“企业流程、知识库、Agent 编排、系统集成、治理、商业化和 ROI”；
- Notion 继续采用高价值信息筛选原则，不把全部资讯无差别入库；
- 云端 GitHub Actions 作为主要定时保障，本地 macOS `launchd` 作为补充；
- 保持原有通用情报 Pipeline 不变，竞品采购模块采用独立配置和独立输出目录。

```text
data/competitive_procurement/daily_YYYY-MM-DD.md

```

## v1.0.0 — 2026-07-05

### 版本主题

完成 AI/Agent 自动化情报收集系统 MVP，建立从数据源到日报和 Notion 的基础闭环。

### 已实现

- 建立政策监管、Agent 产品、AI Infra、安全合规、投融资和招聘竞品等数据源配置；

- 支持跨来源事件归并和低价值噪音过滤；
- 生成日报 Markdown 和运行 Summary；
- 支持 SQLite、本地导出和 Notion 同步；
- 建立每日、每周和每月复盘机制；
- 提供 macOS `launchd` 定时任务模板；
- 建立产品规划、解决方案设计、执行节奏和 Notion 配置文档。
