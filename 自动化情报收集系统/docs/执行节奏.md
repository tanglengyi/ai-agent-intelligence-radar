# 执行节奏

## 每天 20 分钟

时间建议：早上 9:30 或晚上 21:30。

1. 运行抓取：

```bash
python3 scripts/intel_radar.py run --enhance --sync-notion --limit 30
```

2. 打开 `reports/YYYY-MM-DD-日报.md`。
3. 只看标题、来源、证据等级和可行动作。
4. 给值得复盘的信息改状态：
   - `未复盘`
   - `已读`
   - `进入周复盘`
   - `进入需求池`
   - `进入面试素材`
5. 每条最多写三句话：
   - 发生了什么？
   - 为什么重要？
   - 对我做产品/面试/判断公司有什么影响？

## 每周 60-90 分钟

时间建议：周日晚上。

1. 运行周报：

```bash
python3 scripts/intel_radar.py weekly
```

2. 回答四个问题：
   - 这周 AI/Agent 最大变化是什么？
   - 哪些方向被政策利空？
   - 哪些方向对 B 端产品经理有机会？
   - 哪些话可以放进面试表达？
3. 把 1-3 条信号转入 `docs/需求池.csv`。
4. 更新一条面试表达：

模板：

> 我不是只看新闻标题，我会把信号拆成来源、证据等级、受影响对象和产品传导。比如某类 Agent 能力调整，我会先看政策时间点，再看平台公告，再看招聘 JD 和竞品产品入口变化，最后判断它对 B 端权限、审计、人工接管和数据边界的影响。

## 每月半天

1. 从周报里选一个主题。
2. 写成一篇 1200-2000 字复盘：
   - 背景
   - 关键证据
   - 产品判断
   - B 端机会
   - 风险与反例
   - 我会怎么做产品
3. 推荐选题：
   - 拟人化智能体下架后，B 端 Agent 产品机会在哪里
   - 模型网关/API 中转站是不是好生意
   - 企业知识库为什么从问答走向流程 Agent
   - AI 产品经理如何判断公司是在做产品还是追热点

## macOS 定时运行

先确认脚本可执行：

```bash
chmod +x scripts/run_daily.sh scripts/run_weekly.sh
```

如果想让定时任务自动做 LLM 摘要和 Notion 同步，在 `.env` 里打开：

```bash
ENABLE_LLM=1
ENABLE_NOTION_SYNC=1
```

安装每日任务：

```bash
cp templates/com.local.ai-intel-radar.daily.plist ~/Library/LaunchAgents/
launchctl load ~/Library/LaunchAgents/com.local.ai-intel-radar.daily.plist
```

查看是否安装成功：

```bash
launchctl list | grep ai-intel-radar
```

卸载：

```bash
launchctl unload ~/Library/LaunchAgents/com.local.ai-intel-radar.daily.plist
rm ~/Library/LaunchAgents/com.local.ai-intel-radar.daily.plist
```

## 什么时候该做什么

| 时间 | 动作 | 产物 |
| --- | --- | --- |
| 每天 | 抓取、扫摘要、标记状态 | 日报、Notion 新增项 |
| 每周 | 聚合同向信号、写产品判断 | 周报、需求池更新 |
| 每月 | 选题沉淀、形成面试资产 | 文章、作品集案例 |
| 面试前 3 天 | 筛选 B 端机会和监管变化 | 面试话术卡片 |
| 投递公司前 | 查公司招聘和产品更新 | 定制化面试问题 |
