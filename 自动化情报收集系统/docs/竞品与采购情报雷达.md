# AI 竞品与采购情报雷达

## 1. 模块目标

该模块不是继续扩大新闻收藏量，而是把外部信号转化为四类可以执行的资产：

1. 竞品功能、价格与商业模式变化；
2. 客户真实采购需求、预算和采购阶段；
3. 目标客户的战略方向、经营压力和数字化投入；
4. 可进入产品规划、销售机会、客户拜访和面试表达的判断。

入口文件：

- `config/competitive_procurement.json`：十类信号、关键词、权重和数据源配置；
- `src/competitive_procurement.py`：采集、采购阶段识别、预算提取、评分、日报和 Notion 发布；
- `data/competitive_procurement/`：每日结果目录。

## 2. 十类数据源框架

| 数据来源 | 可以发现什么 | 模块处理方式 |
| --- | --- | --- |
| 政策和监管变化 | 强制性需求、合规预算、行业准入门槛 | 识别政策、标准、备案、实施日期和受影响行业 |
| 行业研究报告 | 市场规模、增长趋势、行业结构 | 记录市场口径、CAGR、细分赛道和研究机构 |
| 竞品官网和报价 | 功能边界、商业模式、价格区间 | 监控价格、套餐、企业版、API、限流和 SLA 变化 |
| 招聘信息 | 企业正在建设哪些能力 | 从 JD 提取岗位能力、技术栈、组织投入和区域布局 |
| 招标采购平台 | 真实预算、采购标准、项目密度 | 识别采购意向、招标、中标、合同、预算和供应商 |
| 行业会议和展会 | 热点方向、供应商格局、客户关注点 | 汇总高频议题、参展厂商、客户行业和生态关系 |
| 客户公开年报 | 战略方向、数字化投入、经营压力 | 提取 AI 战略、数字化预算、资本开支和管理层表述 |
| 搜索趋势和社交内容 | 需求热度、用户讨论焦点 | 作为早鸟弱信号，要求用其他来源二次验证 |
| 渠道和合作伙伴 | 一线需求、区域差异、生态分工 | 记录合作对象、区域、行业、渠道政策和交付边界 |
| 技术发展 | 原来做不了、现在可以做的场景 | 把模型、Agent、推理和开源变化翻译为场景与成本变化 |

## 3. 每日运行

在项目目录执行：

```bash
python3 src/competitive_procurement.py --date today
```

限制单源条数并只输出 Top 20：

```bash
python3 src/competitive_procurement.py \
  --date today \
  --limit-per-source 5 \
  --top-n 20
```

每日生成：

```text
data/competitive_procurement/signals_YYYY-MM-DD.jsonl
data/competitive_procurement/daily_YYYY-MM-DD.md
data/competitive_procurement/summary_YYYY-MM-DD.json
```

## 4. 采购信号结构

每条信号包括：

- `signal_category`：十类信号之一；
- `procurement_stage`：`intent`、`tender`、`award`、`partnership` 或 `unknown`；
- `budget_text`：原文中的金额表达；
- `budget_cny`：统一成 CNY 的金额，便于排序；
- `tracked_companies`：命中的竞品或供应商；
- `matched_keywords`：触发规则的关键词；
- `score`：0-10 分；
- `confidence`：`high`、`medium`、`low`；
- `recommended_action`：下一步建议动作。

评分主要考虑：数据源证据等级、类别权重、是否出现明确采购阶段、是否有预算、预算规模、目标公司命中和关键词密度。

## 5. Notion 同步

该模块把每日竞品采购报告发布为 Notion 父页面下的子页面。

在 `.env` 或 GitHub Actions Secrets 中配置：

```dotenv
NOTION_TOKEN=secret_xxx
NOTION_COMPETITIVE_PARENT_PAGE_ID=xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
```

如果没有单独设置 `NOTION_COMPETITIVE_PARENT_PAGE_ID`，程序会回退使用：

```dotenv
NOTION_PARENT_PAGE_ID=xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
```

手动同步：

```bash
python3 src/competitive_procurement.py \
  --date today \
  --notion-parent-page-id YOUR_PAGE_ID
```

需要先把 Notion Integration 授权给目标父页面，否则 Notion API 会拒绝创建页面。

## 6. 每日定时任务

### GitHub Actions

仓库新增 `.github/workflows/daily-ai-intelligence.yml`：

- 每天北京时间 08:30 自动运行；
- 也支持在 Actions 页面手动触发；
- 先运行单元测试；
- 再运行通用情报 Pipeline；
- 最后运行竞品采购雷达；
- 日报和 JSONL 作为 GitHub Actions Artifact 保留 30 天；
- 配置 Notion Secrets 后自动写入 Notion。

建议在仓库 `Settings -> Secrets and variables -> Actions` 配置：

| Secret | 必需性 | 作用 |
| --- | --- | --- |
| `NOTION_TOKEN` | Notion 同步必需 | Notion Integration Secret |
| `NOTION_COMPETITIVE_PARENT_PAGE_ID` | 竞品日报同步建议配置 | 竞品采购日报父页面 |
| `NOTION_PARENT_PAGE_ID` | 可选 | 通用日报父页面，也可作为竞品日报回退页面 |
| `NOTION_DATABASE_ID` | 可选 | 通用情报事件数据库 |
| `RSSHUB_BASE_URL` | 可选 | 自建 RSSHub 地址；未配置使用公共实例 |

### macOS 本地任务

现有 `launchd` 模板每天 09:30 运行 `scripts/run_daily.sh`。每日脚本已同时调用通用情报流程和竞品采购雷达。

本地任务只有在安装 plist 后才会生效：

```bash
cp templates/com.local.ai-intel-radar.daily.plist ~/Library/LaunchAgents/
launchctl load ~/Library/LaunchAgents/com.local.ai-intel-radar.daily.plist
```

模板里包含本机绝对路径，换电脑或调整项目目录时需要修改路径。

## 7. 数据质量原则

1. 招聘、社媒和搜索结果只能视为弱信号；
2. 招标、中标、合同、年报和官方公告优先进入机会清单；
3. 同一结论至少由两个独立来源交叉验证；
4. 金额识别目前是规则解析，复杂表格、附件 PDF 和币种换算需要人工复核；
5. RSSHub 和搜索索引可能存在延迟或失效，应定期检查源健康度；
6. Notion 是知识资产库，不应把所有原始资讯无差别写入。
