# 产品规划

## 产品定位

把 AI/Agent 分散信息流变成产品经理可复用的判断资产。系统服务三个场景：

1. 求职面试：形成“我如何判断行业变化”的表达。
2. 竞品研究：追踪平台、模型、Agent、Infra、合规方向的动作。
3. 作品集沉淀：把周复盘变成文章、产品方案、需求池和案例。

## 用户画像

- 当前角色：AI 产品经理求职者 / 转型者。
- 核心痛点：信息很多但没有证据链，容易被情绪型内容带节奏。
- 成功标准：每周能产出 1 条结构化产品判断，每月能沉淀 1 篇面试资产或作品集材料。

## 核心流程

1. 配置信息源：官方、平台、媒体、研究库、社交网络、招聘。
2. 自动抓取：RSS 优先；无 RSS 的网页做页面监控。
3. 去重入库：用链接指纹去重，避免重复收藏。
4. 自动标签：类型、方向、影响对象、证据等级、影响判断。
5. 复盘判断：每日轻扫，周度归纳，月度沉淀。
6. 反哺需求池：把高频信号转成产品机会、面试表达或作品集选题。

### V0.2 已完成

## 衡量指标

- 每日新增有效信息数：10-30 条即可，不追求越多越好。
- 每周结构化判断数：至少 3 条。
- 每月可公开/可面试资产：至少 1 篇。
- 信息源健康度：官方/公司公告占比不低于 40%。
- 需求转化率：每周至少 1 条情报进入需求池或面试素材库。
